## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebUI request ŌåÆ Gateway route | Untrusted JSON crosses into authenticated Gateway route. |
| Gateway ŌåÆ CLI socket RPC | Gateway forwards metadata requests to user-owned CLI only. |
| CLI socket event ŌåÆ WebUI relay | CLI-originated projection must be scoped to the owning user. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-S | Spoofing | `/acp/agent-teams` routes | mitigate | Apply `requireAuth` and `getUserId` before every route handler. |
| T-03-02-T | Tampering | create/list/get params | mitigate | Validate machine ownership and reject client-supplied lifecycle/MCP truth. |
| T-03-03-I | Information Disclosure | Gateway route body/logs | mitigate | Recursively reject forbidden plaintext and secret keys before RPC; logs include only IDs/status/counts/error code. |
| T-03-04-I | Information Disclosure | `agent-teams:changed` relay | mitigate | Use `record.userId` from CLI socket registry; never global broadcast. |
| T-03-05-R | Repudiation | TeamRouter RPC | accept | Phase 1 follows existing requestId logging; detailed audit events belong to later lifecycle/security phases. |
</threat_model>
