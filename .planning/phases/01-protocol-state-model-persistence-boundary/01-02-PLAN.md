<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI SQLite rows → Gateway-facing projection | CLI-local rows may contain `body_local_json`; projection must strip plaintext content before RPC/Gateway/WebUI. |
| Existing WAL database → AgentTeamStore | Agent Team tables share the WAL database file; migrations must not drop or corrupt ordinary session tables. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-I | Information Disclosure | `projection-builder.ts` | mitigate | Build projection only from allowed IDs/status/counts/timestamps/source refs/errors and assert no forbidden keys before returning. |
| T-02-02-T | Tampering | `AgentTeamStore.createAgentTeam` | mitigate | Store owns lifecycle defaults; caller supplies metadata only, not durable lifecycle/MCP truth. |
| T-02-03-T | Tampering | WAL migrations | mitigate | Add Agent Team tables through existing `MIGRATIONS`/`schema_version`; tests cover existing session schema remains present. |
| T-02-04-D | Denial of Service | recursive boundary assertion | accept | Phase 1 tests cover nested objects; payloads are local projections and small metadata objects. |
</threat_model>
