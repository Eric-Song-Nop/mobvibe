<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI-local body rows → projection counts | Mailbox/task rows can contain local plaintext; projection must output metadata only. |
| persisted JSON → TypeScript objects | JSON columns are untrusted persisted data and require narrowing before projection. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-I | Information Disclosure | mailbox projection | mitigate | Count unread/wake/source refs only; never map `body_local_json` to Gateway projection. |
| T-04-02-I | Information Disclosure | task projection | mitigate | Count status/dependencies/source refs only; never expose title/body/description. |
| T-04-03-T | Tampering | JSON parsing | mitigate | Parse JSON into `unknown`, validate arrays/source-ref shape, and fall back to safe empty metadata on invalid local rows with explicit error handling. |
| T-04-04-I | Information Disclosure | summary refs | mitigate | Persist/project source refs and status only; summary body is not a Phase 1 Gateway-facing field. |
</threat_model>
