<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP tool args → MailboxService | Agent-controlled recipient/body data enters durable local store. |
| CLI-local mailbox body → Gateway projection | Plaintext body must not cross to Gateway/WebUI projection. |
| Durable write → wake metadata | Delivery and wake state are separate facts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Spoofing | `mobvibe_team_send_message` | mitigate | Sender comes only from router-bound caller context; reject/ignore `fromMemberId` args. |
| T-02-03-02 | Information disclosure | `projection-builder.ts` / `socket-client.ts` | mitigate | Projection selects metadata only and runs `assertGatewayFacingAgentTeamPayload`; tests serialize payload and assert no body text. |
| T-02-03-03 | Information disclosure | logging | mitigate | Log message ids, team ids, sender/recipient ids, wake status; never log `message`, `summary`, `body_local_json`, secrets, DEK, or tokens. |
| T-02-03-04 | Tampering | broadcast recipient resolution | mitigate | Resolve recipients from durable team members, exclude sender for `*`, reject unknown names/ids before insert. |
| T-02-03-05 | Elevation of privilege | `mobvibe_team_send_message` policy | mitigate | D-07/D-08 allow all bound team members and no team-tool confirmation; enforce D-10 structural validation and keep D-09 ordinary ACP permissions unchanged. |
</threat_model>
