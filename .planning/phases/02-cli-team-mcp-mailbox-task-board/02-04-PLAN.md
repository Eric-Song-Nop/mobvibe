<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI-local mailbox body → ordinary ACP session prompt | Plaintext intentionally enters recipient session history, which is E2EE/WAL-owned. |
| Ordinary session event → Gateway | Encrypted session event path may carry injected content; team projection must not. |
| Member activity → leader wake | Bad guard can create wake loops or message storms. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Information disclosure | `TeamRuntime.wakeMember` | mitigate | Inject plaintext only via ordinary session prompt/WAL path; never include body in `agent-teams:changed`. |
| T-02-04-02 | Denial of service | idle notification guard | mitigate | Wake leader only when all non-leader members are settled and avoid recursive wake-on-wake loops. |
| T-02-04-03 | Repudiation | read-and-mark | mitigate | Atomic read/mark preserves `read_at`, `wake_status`, source refs, and delivered session ids for auditability. |
| T-02-04-04 | Information disclosure | logs | mitigate | Log only ids/status/errors; never log injected message body, prompt text, secrets, tokens, master secret, or DEK. |
</threat_model>
