<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ACP mcp messages → CLI router | Agent-controlled JSON-RPC method/params enter local runtime. |
| Router binding → tool handlers | Caller identity must be transport-derived before durable writes. |
| Session creation → ACP backend | Team tools must be injected only for explicit team sessions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Spoofing | `TeamMcpRouter` | mitigate | Bind caller from `serverId`/connection mapping; reject or ignore `fromMemberId` in tool args. |
| T-02-02-02 | Elevation of privilege | `AcpConnection.createSession` | mitigate | Ordinary tests assert `mcpServers: []`; team declaration requires explicit team context. |
| T-02-02-03 | Information disclosure | router/tool logs | mitigate | Log request ids, team/member ids, tool names, status only; never plaintext mailbox/task bodies or secrets. |
| T-02-02-04 | Repudiation | MCP readiness | mitigate | Persist readiness phase transitions in `agent_team_mcp_status` with `updated_at` and error metadata. |
| T-02-02-05 | Tampering | spawn/rename/shutdown intent facts | mitigate | Persist only router-bound requested-by identity and sanitized payload; do not execute lifecycle side effects in Phase 2. |
| T-02-02-06 | Elevation of privilege | team-tool policy | mitigate | D-07/D-08 intentionally remove leader-only and team-tool confirmation gates; compensate with D-10 structural validation and D-09 unchanged ordinary ACP permission handling. |
</threat_model>
