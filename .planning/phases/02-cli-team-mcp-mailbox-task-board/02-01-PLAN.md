<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ACP backend capabilities → CLI team validation | Backend-reported capabilities decide whether autonomous team tools are exposed. |
| Team declaration builder → ACP session/new payload | A bad declaration can leak team tools into ordinary sessions. |
| Fallback bridge decision → local agent config | Fallback must remain per-session and never mutate global MCP configuration. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Spoofing | `team-capability.ts` declaration helpers | mitigate | Build caller server id from component-owned `agentTeamId/memberId`; do not accept caller identity from tool args. |
| T-02-01-02 | Elevation of privilege | `acp-connection.ts` ordinary path | mitigate | Keep ordinary `session/new` and `session/load` tests asserting `mcpServers: []`. |
| T-02-01-03 | Elevation of privilege | bridge eligibility | mitigate | `resolveTeamMcpTransport` permits only per-session bridge capability; no global config mutation code path. |
| T-02-01-04 | Information disclosure | logs/errors | mitigate | Log backend ids and capability booleans only; never log mailbox/task body, prompt, provider token, master secret, DEK, or raw env values. |
</threat_model>
