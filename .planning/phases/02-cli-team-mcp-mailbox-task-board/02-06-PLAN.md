<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Native ACP capability → bridge fallback | Missing native support must not cause unsafe global config mutation. |
| Bridge process/config → ACP backend session | Fallback must be scoped to one team member session. |
| Bridge tool call → team handlers | Tool caller identity must still be server/session-bound, not args-bound. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06-01 | Elevation of privilege | `team-bridge-stdio.ts` | mitigate | Build only per-session bridge declarations; tests assert no global MCP config writes. |
| T-02-06-02 | Spoofing | bridge caller context | mitigate | Bridge args/env are generated by CLI and map to one `agentTeamId/memberId`; handlers still reject caller override args. |
| T-02-06-03 | Information disclosure | bridge logs/process args | mitigate | Do not place mailbox/task bodies or secrets in bridge env/args/logs; only ids and auth-free routing metadata. |
| T-02-06-04 | Denial of service | unsupported backends | mitigate | Fail fast with `CAPABILITY_NOT_SUPPORTED` instead of creating a session that cannot receive tools. |
</threat_model>
