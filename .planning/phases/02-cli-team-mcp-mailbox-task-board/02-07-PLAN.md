<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ACP backend → AcpConnection MCP callbacks | Untrusted agent/backend MCP messages enter CLI runtime. |
| AcpConnection → TeamMcpRouter | Normalized MCP message data becomes authenticated team/member caller context. |
| TeamToolHandlers → AgentTeamStore SQLite | Tool args mutate durable mailbox/task facts. |
| AgentTeamStore → Gateway-facing projection | CLI-local bodies must not cross into Gateway payloads. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07-01 | Spoofing | ACP MCP tool-call adapter | mitigate | Bind caller exclusively by `serverId` parsed by `TeamMcpRouter`; tests pass spoofed args and assert persisted identity remains bound member. |
| T-02-07-02 | Elevation of privilege | stdio bridge selection | mitigate | Reject `stdio_bridge` in `team-capability.ts` until an executable per-session MCP stdio server exists; never mutate global MCP config. |
| T-02-07-03 | Tampering | task/mailbox tool args | mitigate | Preserve existing `TeamToolHandlers`, `MailboxService`, and `TaskBoardService` validation; unknown MCP message shapes fail closed. |
| T-02-07-04 | Information disclosure | MCP logging/projection | mitigate | Do not log tool args/message bodies; projection assertions serialize `AgentTeamSummary` and verify mailbox/task plaintext/body keys are absent. |
| T-02-07-05 | Denial of service | malformed MCP callback messages | mitigate | Normalize with bounded `unknown` parsers and structured errors; disconnect clears binding. |
</threat_model>
